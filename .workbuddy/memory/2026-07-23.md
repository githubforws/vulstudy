# 2026-07-23 工作日志

## 修复：环境编排启动时 Docker 网络子网冲突

**问题**：启动编排场景时 docker-compose up 报错 `Pool overlaps with other one on this address space`

**根因**：编排创建/保存时（`download_official_website_layout` 及 `create`）会通过 Docker SDK 预创建 Docker 网络（占用用户指定的子网如 172.1.1.0/24），而 `run_layout` 中 `build_yml()` 生成的 docker-compose.yml 使用相同子网，导致 docker-compose 创建网络时子网冲突。

**修复**：在 `vulfocus-api/layout_image/views.py` 中：
1. 添加了 `_cleanup_layout_docker_networks()` 方法，通过 `LayoutServiceNetwork` 查找预创建的网络并移除
2. 在 `run_layout` 中 docker-compose up 之前调用该方法清理网络
3. 添加了 logging 支持

## 修复：内网 icc=false 导致 DMZ 跳板无法访问内网目标

**问题**：多网络隔离模式下，所有网络（含 DMZ 和内网）均设置 `icc=false`，导致 ThinkPHP (DMZ) 无法通过内网 IP 访问 WordPress (Internal)，无法模拟"外网 → DMZ 跳板 → 内网目标"的经典渗透路径。

**根因**：`build_yml()` 中所有网络统一使用 `driver_opts: {"com.docker.network.bridge.enable_icc": "false"}`，阻止同 bridge 内容器间通信。

**修复**：修改 `build_yml()` 网络配置逻辑：
- **内网 (internal)**：不设 icc=false，允许容器间通信实现渗透路径；保留 `internal: true` 阻止外网访问
- **DMZ**：保持 icc=false，仅通过端口映射对外暴露
- 跨场景隔离由 Docker 不同 bridge 网络天然保证，不受影响

## 修复：创建计时模板 500 错误 — 前后端字段名不匹配

**问题**：`POST /timetemp/` 报 `KeyError: 'desc'`，500 Internal Server Error

**根因**：前端 Vue 3 重写后字段名改变（`time_desc` / `timer_minutes`），后端 `CreateTimeTemplate.create` 使用直接键值访问 `request.data['desc']` / `request.data['time_range']` 等旧字段名，且 `time_img_type`、`ilist`、`template_pattern`、`imageName` 多个字段前端不再发送，也会逐个触发 KeyError

**修复**：`vulfocus-api/dockerapi/views/time_views.py` 的 `create` 方法：
- 全部改为 `request.data.get('key', default)` 防御式访问
- 字段名对齐前端：`desc` → `time_desc`，`time_range` → `timer_minutes`（并兼容旧 `time_range`）
- 缺失字段给空字符串默认值
- 移除 `template_pattern` 不为空的强制校验（前端未提供此字段）
- **补充修复**：`template_pattern` 是 `IntegerField(default=1)`，默认值改为 `None or 1`，避免传空字符串导致 `ValueError: Field 'template_pattern' expected a number but got ''`

## 修复：启动盲盒 500 — TimeMoudelSet.create 同样硬读 request.data

**问题**：`POST /time/` 报 `KeyError: 'time_range'`，前端只发 `{temp_id: "xxx"}`

**根因**：与计时模板创建同一个模式 — `TimeMoudelSet.create` 使用 `request.data['time_range']` 硬读，但前端 Vue 3 重写后不再发送此字段

**修复**：`time_range` 应从模板记录读取，非请求参数。改为先查 `TimeTemp` 取 `time_range`，模板不存在则返回错误
