## 修复：axios 拦截器漏检业务错误码导致前端误报成功

**问题**：后端返回 HTTP 200 + `code: 2001`（业务错误），前端 axios 拦截器只判断 HTTP 状态码，`code: 2001` 被当作成功响应，导致 `ElMessage.success` 误弹

**修复**：`request.js` 响应拦截器增加业务 `code` 检查：`code` 非 200 且非空时 reject 为错误，统一触发 ElMessage.error

## 修复：dashboard 停止计时后旧倒计时滞留 + 无轮询

**问题**：Admin 停止→重启计时后，dashboard 仍显示旧倒计时；且 `fetchTimerMode` 只在 mount 时调一次

## 修复：R.ok() 返回 status 而非 code，导致前端手动检查误判

**问题**：`R.ok()` 返回 `{status: 200, msg: "OK"}` 没有 `code` 字段，而 `network/index.vue` 等页面检查 `res.data.code !== 200` —— `code` 为 `undefined`，永远进入错误分支，导致 `ElMessage.error("OK")` 出现红色错误提示

**修复**：
- `request.js` 拦截器：`data.code` 改为 `data.code ?? data.status`，同时兼容两种格式
- `network/index.vue`、`setting.vue`（3处）、`images.vue`（1处）：删除冗余的 `data.code !== 200` 手动检查（拦截器已全局处理）

**修复**：`dashboard/index.vue`
- `fetchTimerMode` 增加"无活跃计时"分支：关闭通知、清 interval
- 创建新通知前先关闭旧通知（防轮询堆叠）
- `onMounted` 增加 `setInterval(fetchTimerMode, 10000)` 10 秒轮询
- `onBeforeUnmount` 增加 `timerModePoll` 和 `countdownTimer` 清理
